# OSI 第一层：物理层（Physical Layer）

> 分类：计算机网络
> 标签：物理层, 传输介质, 编码方式, 调制解调, 复用技术, 比特流, 网线, 光纤

## 面试精答

> 适合面试时口述的简洁版本（2-3分钟能说完的要点）

物理层是 OSI 模型的最底层，负责在物理介质上传输**原始比特流（0 和 1）**。它不关心这些比特代表什么含义，只关心"怎么把 0 和 1 从一端送到另一端"。

**物理层解决的核心问题**：
1. **用什么介质传输？** — 双绞线、光纤、无线电波
2. **0 和 1 怎么变成信号？** — 电平高低、光强弱、频率变化（编码和调制）
3. **传输多快？** — 带宽（bps）、波特率
4. **传输方向？** — 单工、半双工、全双工

**关键设备**：中继器（Repeater）、集线器（Hub）。这两个设备的本质都是**信号放大器**——数字信号在传输过程中会衰减和失真，中继器/集线器负责把信号"还原"后继续传输。

**物理层不负责的内容**：不负责纠错（那是数据链路层的事）、不负责寻址（没有地址概念）、不负责路由（不知道数据往哪走）。它就是一个"搬运工"，把比特流从 A 点搬到 B 点。

## 原理深入详解

> 深入底层原理，理解"为什么"

### 一、传输介质

物理层首先要解决"数据在什么东西上跑"的问题。

#### 1. 导引型传输介质（有线的）

**双绞线（Twisted Pair）**：

为什么叫"双绞"——两根铜线绞在一起。绞的目的是抵消外部电磁干扰：两根线靠得近，受到的干扰几乎相同，接收端做差分运算时干扰被抵消。

```
双绞线的类别：

Cat5  — 100 MHz — 最高 100 Mbps（百兆网）
Cat5e — 100 MHz — 最高 1 Gbps（千兆网，家用主流）
Cat6  — 250 MHz — 最高 10 Gbps（短距离）
Cat6a — 500 MHz — 最高 10 Gbps（100米）
Cat7  — 600 MHz — 最高 10 Gbps（屏蔽双绞线）
```

你家里路由器连电脑的那根网线，大概率是 Cat5e 或 Cat6 的双绞线。

**RJ-45 接口的线序**：网线两头有 8 根线芯（4 对双绞线），有 T568A 和 T568B 两种排线标准。千兆网（1000BASE-T）用到了全部 4 对线同时收发。

**光纤（Optical Fiber）**：

用光脉冲（有光 = 1，无光 = 0）在玻璃纤维中传输数据。核心优势：带宽极大、抗电磁干扰、传输距离远。

```
光纤的分类：

单模光纤（Single-Mode）
  - 纤芯极细（~9 μm），只允许一束光直线传播
  - 传输距离远（几十~上百公里）
  - 用于骨干网、城域网、长距离通信
  - 激光光源，成本高

多模光纤（Multi-Mode）
  - 纤芯较粗（~50/62.5 μm），允许多束光以不同角度传播
  - 传输距离短（几百米~2公里）
  - 用于数据中心内部、楼宇之间
  - LED 光源，成本低
```

**为什么光纤比铜线快**：光的频率远高于电信号频率，所以光可以承载更多信息。一根单模光纤的理论带宽接近无限，实际瓶颈在于两端的激光器和接收器的速度。

**同轴电缆（Coaxial Cable）**：

中心铜导体 + 绝缘层 + 网状屏蔽层 + 外皮。比双绞线抗干扰能力强，但比光纤笨重。现在主要用于有线电视网络（Cable TV）和早期的以太网（10BASE2），在数据通信中已基本被双绞线和光纤取代。

#### 2. 非导引型传输介质（无线的）

| 类型 | 频段 | 传输距离 | 典型应用 |
|---|---|---|---|
| **无线电波** | 全向传播 | 几米~几十公里 | WiFi、蜂窝网络、蓝牙 |
| **微波** | 直线传播 | 几十公里（视距内） | 卫星通信、地面微波中继 |
| **红外线** | 直线传播 | 几米 | 遥控器、红外数据传输 |
| **激光** | 直线传播 | 几公里 | 自由空间光通信（实验性） |

**WiFi 为什么是无线电波而不是光纤**：WiFi 的核心需求是"移动性"——设备需要随时随地接入。无线电波可以穿墙、覆盖一定区域，虽然带宽不如光纤，但胜在灵活。

**5G 为什么比 4G 快**：5G 使用了更高的频率（毫米波，24-100 GHz），频率越高能承载的信息越多，但传播距离越短、穿透能力越差。所以 5G 需要部署更多更密的小型基站。

### 二、数据怎么变成信号：编码与调制

计算机里的数据是 0 和 1（数字信号），但物理介质上传输的是模拟信号（电压、光强、电磁波频率）。**编码和调制就是数字信号和模拟信号之间的桥梁。**

#### 编码（数字信号 → 数字信号）

把 0 和 1 转换成适合在数字信道上传输的电平信号。

**NRZ（Non-Return to Zero，不归零编码）**：

```
数据:  1  0  1  1  0  0  1
电平:  高 低  高 高 低  低  高
       ┌─┐   ┌─┬─┐     ┌─
       │ │   │ │ │     │
       │ │   │ │ │     │
  ─────┘ └───┘ └─└─────┘

问题：连续的 0 或 1 时电平不变，接收方无法区分"没有信号"和"连续的 0"
```

**曼彻斯特编码（Manchester Encoding）**：

```
数据:    1      0      1      0
电平:  高→低   低→高  高→低   低→高
       ┌─┐   ┌─┐   ┌─┐   ┌─┐
       │ └─┐─┘ └─┐─┘ └─┐─┘ │
  ─────┘   └─────┘   └─────┘

每个比特中间都有一次跳变：
  下降沿（高→低）= 1
  上升沿（低→高）= 0

以太网（10BASE-T）使用的就是曼彻斯特编码
优点：每个比特都有跳变，接收方可以从中提取时钟信号（自同步）
缺点：频率翻倍（10 Mbps 数据需要 20 MHz 带宽）
```

**4B/5B 编码**：

```
每 4 位数据映射到 5 位编码，保证编码后的序列不会出现超过 3 个连续 0
用于 100BASE-TX（百兆以太网）

效率：80%（5 位编码传 4 位数据）
```

#### 调制（数字信号 → 模拟信号）

把 0 和 1 "搭载"到载波信号上，在模拟信道（如电话线、无线电）上传输。

载波是一个固定频率的正弦波，有三个可以调的参数：**振幅（A）、频率（f）、相位（φ）**。

```
调幅（AM，Amplitude Shift Keying）：
  振幅大 = 1，振幅小 = 0
       ┌─┐     ┌─────┐
       │ │     │     │
  ─────┘ └─────┘     └──

调频（FM，Frequency Shift Keying）：
  频率高 = 1，频率低 = 0
       ╱╲╱╲  ╱╲╱╲╱╲  ╱╲
      ╱  ╲╱  ╲    ╱  ╲╱
     ╱        ╲╱

调相（PSK，Phase Shift Keying）：
  相位跳变 = 1，相位不变 = 0
       ┌─┐  ╲╱  ┌─┐
       │ │ ╱ ╲  │ │
  ─────┘ └╱   ╲─┘ └──
```

**现代调制技术更复杂**：比如 16-QAM 同时调振幅和相位，一个符号可以表示 4 个比特，大幅提升传输效率。

### 三、信道容量：传输速率的上限

**奈氏准则（Nyquist Theorem）**：

在无噪声的理想信道中，最大传输速率 = 2W log₂(V) bps

```
W = 信道带宽（Hz）
V = 每个信号离散状态数

例如：电话线带宽 4000 Hz，使用 16 种不同的信号状态（V=16）
最大速率 = 2 × 4000 × log₂(16) = 2 × 4000 × 4 = 32,000 bps = 32 Kbps
```

**香农定理（Shannon Theorem）**：

在有噪声的实际信道中，最大传输速率 = W log₂(1 + S/N) bps

```
W = 信道带宽（Hz）
S/N = 信噪比（Signal-to-Noise Ratio）

信噪比通常用分贝（dB）表示：dB = 10 × log₁₀(S/N)
例如：信噪比 30 dB → S/N = 1000

带宽 4000 Hz，信噪比 30 dB：
最大速率 = 4000 × log₂(1 + 1000) = 4000 × 9.97 ≈ 39.86 Kbps
```

**两者的关系**：奈氏准则给出了理想情况的理论上限，香农定理给出了有噪声的实际上限。实际速率不可能超过两者中的较小值。

### 四、多路复用技术

一条物理线路的带宽很大，怎么让多个信号共享同一条线路？这就是多路复用。

```
                        复用器                分用器
信号1 ──→ ┌──┐                    ┌──┐ ──→ 信号1
信号2 ──→ │  │ ══════════════════ │  │ ──→ 信号2
信号3 ──→ │  │   共享物理链路      │  │ ──→ 信号3
信号4 ──→ └──┘                    └──┘ ──→ 信号4
```

| 复用方式 | 原理 | 应用场景 |
|---|---|---|
| **FDM（频分复用）** | 每个信号占用不同的频率段 | 有线电视、FM 广播、第一代移动通信 |
| **TDM（时分复用）** | 每个信号占用不同的时间段（轮流用） | 电话网络（E1/T1 线路）、2G 移动通信 |
| **WDM（波分复用）** | 光纤上不同波长的光同时传输（光的 FDM） | 光纤骨干网（一根光纤同时传几十~上百个波长） |
| **CDMA（码分复用）** | 每个信号用不同的编码，数学上正交可分离 | 3G 移动通信、GPS |
| **OFDM（正交频分复用）** | 把高速数据分散到多个正交子载波上 | WiFi、4G LTE、5G |

**WDM 的威力**：一根单模光纤用 WDM 技术可以同时传输 80~160 个波长，每个波长跑 100 Gbps，总带宽达到 8~16 Tbps。这就是海底光缆能承载全球互联网流量的原因。

### 五、物理层设备

#### 中继器（Repeater）

```
信号传输过程中会衰减和失真：

发送端: ┌─┐   ┌───┐       ┌─┐
        │ └───┘   └───────┘ └──
                                    传输距离远后
接收端: ┌┐ ┌┐ ┌┐─┐ ┌┐  ┌┐       ← 信号失真，无法辨认
        │└─┘└─┘└─┘ └──┘└─

加中继器后：
发送端 ────→ 中继器 ────→ 接收端
              │
          信号被还原放大
          （重新整形、重新定时）
```

中继器只有 2 个端口，工作在物理层，只是把收到的信号重新生成后转发出去。它不理解任何帧结构、地址信息。

#### 集线器（Hub）

集线器本质是**多端口中继器**。从一个端口收到信号后，**广播到所有其他端口**。

```
集线器的工作方式：

端口1 收到数据 → 广播到 端口2、端口3、端口4

          ┌───── 端口2（收到数据）
          │
端口1 ────┤──── 端口3（收到数据）
          │
          └──── 端口4（收到数据）
```

**集线器的问题**：
1. **所有设备共享带宽**：100 Mbps 的 Hub 接 10 台电脑，每台实际只有 10 Mbps
2. **冲突域不隔离**：两台电脑同时发送数据就会冲突（CSMA/CD 协议处理）
3. **安全性差**：发给电脑 B 的数据，电脑 C、D 也能收到（只是网卡丢弃了不是给自己的帧）

这就是为什么集线器在现代网络中已经被交换机完全取代。

## 常见追问与应对

**Q1: 为什么网线最大传输距离是 100 米？**
> 双绞线的信号衰减和延迟限制了传输距离。Cat5e 在 100 米时信号衰减到刚好能被接收端正确识别的临界点。超过 100 米，信号失真严重导致误码率急剧上升。如果需要更远距离，中间加交换机或改用光纤。

**Q2: 光纤这么好，为什么不全部替换成光纤？**
> 成本。光纤本身不贵，但两端的光模块（光/电转换器）成本高。而且光纤不能弯折太厉害（会破坏全反射），施工难度大。所以通常是骨干网用光纤，最后一百米（入户）用双绞线。FTTH（光纤到户）正在改变这个局面。

**Q3: 全双工和半双工在物理层怎么实现的？**
> 全双工需要两条独立的通信通道——双绞线用不同的线对分别发和收（1000BASE-T 用 4 对线同时双向传输），光纤用两根光纤（一根发一根收）或同一根光纤用两个不同波长的光（WDM）。半双工只有一条通道，收发交替。
